# Product Charter — 区域AI产业企业数据管理平台

## 1) Product Positioning

面向区域AI产业分析师和运营人员，提供企业数据的统一管理、智能筛选与可视化分析能力。帮助用户高效维护数百到上千家企业的结构化信息，快速识别目标企业群体，支撑产业分析、孵化服务、技术赋能和运营活动决策。

我们通过将分散的Excel数据转化为可查询、可追溯、可协作的企业资产库，让区域产业工作从"手工整理"转向"数据驱动"，让每一次筛选和导出都基于最新、最准确的企业画像。

## 2) Brand Keywords

- **高效** — 批量导入、智能去重、快速筛选，将数小时的手工工作缩短到几分钟
- **精准** — 结构化字段、标准化枚举、多维组合筛选，确保每次查询都能精确命中目标企业
- **可信** — 变更日志、审计追踪、权限管控，让数据来源和修改历史清晰可查
- **协作** — 筛选模板共享、运营标签同步、角色权限分工，团队在同一个数据基础上协作
- **洞察** — Dashboard可视化、分布图钻取、趋势分析，让数据讲述产业故事

## 3) Core Problem / JTBD

当区域产业分析师需要为一场"智能制造AI应用挑战赛"筛选目标企业时，他们面临的现实是：企业信息分散在多个Excel表中，字段不统一，更新不及时，每次筛选都需要手工比对行业、AI场景、百度AI使用情况、成熟度等多个维度。这个过程不仅耗时数小时，还容易遗漏或出错。当运营同事催要名单时，分析师往往只能基于"印象"和"上次的表"快速拼凑，无法确保数据的准确性和完整性。更糟糕的是，不同时间导出的名单互相冲突，团队成员各自维护副本，导致数据版本混乱，难以追溯企业的真实变化。

## 4) Goals & Mission

- **Mission:** 将区域AI产业企业数据从"分散的表格"转化为"动态的资产地图"，让每一次产业分析、运营决策和政策制定都基于准确、及时、可追溯的企业画像。

- **Desired Outcomes (descriptive):**
  - 用户能够在几分钟内完成原本需要数小时的企业筛选和名单导出工作
  - 团队在同一个数据源上协作，消除版本冲突和信息孤岛
  - 企业数据的每一次变更都有记录，可以回溯历史状态和修改原因
  - Dashboard和分布图让产业趋势和重点方向一目了然，支撑决策讨论
  - 运营活动的企业参与度和效果可以通过标签和统计持续追踪
  - 数据质量和一致性得到保障，字段枚举标准化，导入校验严格

## 5) Solutions We Own

### 企业数据管理

- **What it solves:** 解决企业信息分散、格式不统一、更新困难、版本混乱的问题
- **Typical path:** 用户通过标准模板批量导入企业数据，系统自动识别字段并完成映射，对重复企业提供去重建议。导入完成后，用户可以在企业详情页中编辑各类字段，所有修改都会被记录到变更日志中。系统支持单条新增、批量更新和字段级合并策略。
- **Outcome for the user:** 用户拥有一个统一、准确、可追溯的企业数据库，告别Excel版本地狱
- **Boundaries:** 不在MVP阶段实现自动化外部数据抓取，仅支持人工导入和维护
- **Guiding principles:** 数据准确性优先 / 导入容错性 / 变更可追溯

### 智能查询与筛选

- **What it solves:** 解决多维条件组合筛选复杂、重复筛选效率低、筛选逻辑难以复用的问题
- **Typical path:** 用户在高级筛选页面通过条件卡片组合多个维度（行业、AI场景、百度AI状态、成熟度、融资轮次等），实时预览匹配企业数量。筛选条件可以保存为模板并设置可见范围，下次使用时一键加载。用户也可以从Dashboard的分布图中直接点击某个区块，自动跳转到对应筛选条件的企业列表。
- **Outcome for the user:** 用户能够快速、精准地锁定任何目标企业群体，并复用常用筛选逻辑
- **Guiding principles:** 条件可视化 / 模板复用 / 实时反馈

### Dashboard与可视化分析

- **What it solves:** 解决产业全局视图缺失、数据分布不直观、趋势难以把握的问题
- **Typical path:** 用户登录后首先看到Dashboard总览，顶部展示企业总数、AI应用企业占比、百度AI使用企业占比等关键指标。中部通过柱状图和饼图展示按行业、AI场景、成熟度的分布情况，用户可以点击图表区块直接跳转到对应企业列表。底部展示最近12个月的新增/更新企业数趋势，帮助识别数据活跃度和产业发展节奏。
- **Outcome for the user:** 用户在几秒钟内就能理解区域AI产业的整体格局和重点方向，并快速钻取到具体企业
- **Guiding principles:** 一目了然 / 可点击钻取 / 时间维度

### 名单导出与运营支撑

- **What it solves:** 解决名单导出格式不统一、字段选择繁琐、运营效果难以追踪的问题
- **Typical path:** 用户在企业列表或筛选结果页面点击"导出"按钮，弹窗中选择需要导出的字段集合（或使用预设模板），选择导出格式（Excel/CSV），确认后系统生成文件并提供下载。运营人员在使用名单后，可以为企业打上运营标签（如"已邀约""已参赛""重点跟进"），这些标签会在系统中持续保留，用于后续筛选和效果评估。
- **Outcome for the user:** 用户能够快速生成符合运营需求的企业名单，并在系统中持续追踪企业的运营参与情况
- **Boundaries:** 不在MVP阶段实现复杂CRM功能，仅保留轻量级运营标签
- **Guiding principles:** 格式灵活 / 标签可追踪 / 审计可查

### 系统配置与字段管理

- **What it solves:** 解决字段和枚举调整依赖开发、业务标准变化响应慢、权限管理不灵活的问题
- **Typical path:** 管理员在系统设置中维护用户账号、角色权限、字段定义和枚举列表。当区域产业分类标准调整时（如新增一个行业类别或AI场景），管理员可以在后台直接添加枚举值，系统会在保存前警告"变更可能影响已有数据统计"。用户管理支持账号创建、启用/停用和角色分配，确保不同角色只接触到与其职责相关的数据和操作。
- **Outcome for the user:** 系统可以随着业务标准和组织需求的变化灵活调整，无需每次都依赖开发改表
- **Boundaries:** MVP阶段仅支持枚举和基础字段配置，不支持完全自定义字段类型
- **Guiding principles:** 配置优于硬编码 / 变更有提示 / 权限细粒度
