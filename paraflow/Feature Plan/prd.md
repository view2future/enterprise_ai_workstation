# PRD — 区域AI产业企业数据管理平台 MVP

## 1. Context

### Alignment with Product Charter

本PRD对应Product Charter中定义的五大解决方案领域的MVP实现,重点聚焦于：

- **企业数据管理**：批量导入、单条编辑、变更日志、去重策略
- **智能查询与筛选**：基础搜索、高级组合筛选、筛选模板保存与复用
- **Dashboard与可视化分析**：总览卡片、分布图、趋势图,支持点击钻取
- **名单导出与运营支撑**：字段自定义导出、运营标签管理
- **系统配置与字段管理**：用户权限管理、字段枚举配置

### Project Type & Iteration Stage

- **项目类型**：全新系统从0到1构建
- **迭代阶段**：MVP第一版,时间周期6-8周
- **核心目标**：替代现有Excel工作流,建立统一数据管理平台

## 2. Objectives & Success Criteria

### Business Objectives

- 上线3个月内,Excel作为主业务工具的使用频次降低80%以上
- 批量更新500家企业的平均耗时 ≤ 30分钟(含导入校验)
- 标准运营名单生成时间从"数小时"降低到 < 10分钟
- 支撑 ≥ 10,000家企业数据,核心查询响应时间 < 3秒

### User Success Criteria

**产业分析师(王雨薇):**
- 能够在30分钟内完成500家企业的批量导入和去重
- 通过组合筛选在5分钟内锁定任何目标企业子集
- Dashboard加载时间 < 3秒,可以一眼看清产业分布
- 关键字段的修改历史100%可追溯

**运营人员(李明浩):**
- 使用预设模板在5分钟内生成活动目标企业名单
- 为企业打标签的操作 < 10秒/条,批量打标 < 1分钟
- 导出Excel的字段和格式符合邮件/微信发送要求

**系统管理员:**
- 新增用户账号 < 2分钟/个
- 调整枚举值(如新增行业类别) < 5分钟,变更即时生效

### Competitive Context

当前内部使用Excel + 手工维护,对标同类型区域产业管理系统(如园区企业管理、招商CRM)。我们的差异化在于：
- 专注AI产业特有字段(AI场景、百度AI使用、成熟度)
- 更灵活的多维筛选和模板复用能力
- 面向分析和运营协作的Dashboard设计

## 3. User Scope & Scenarios

### Target Users

1. **产业分析师(Primary)** — 数据维护和分析的核心用户,使用频率:每日
2. **运营人员(Secondary)** — 活动组织和名单生成的主要用户,使用频率:每周
3. **系统管理员(Admin)** — 账号、权限和配置管理,使用频率:按需

### User Stories

#### 产业分析师

**US-1: 批量导入企业数据**
- As 产业分析师, I want to 使用标准模板批量导入企业列表, so that 我可以把现有Excel快速迁移到系统
- **场景**: 首次上线时导入现有500家企业数据,或每月新增企业批量导入
- **流程**: 下载模板 → 填充数据 → 上传文件 → 字段映射确认 → 处理去重冲突 → 查看导入结果报告
- **成功标准**: 500条数据导入完成时间 < 30分钟,去重准确率 > 95%

**US-2: 企业信息编辑与变更追溯**
- As 产业分析师, I want to 在详情页编辑企业的各类字段并查看修改历史, so that 我可以维护准确数据并追溯变化
- **场景**: 企业完成新一轮融资、AI应用场景变化、百度AI使用状态更新
- **流程**: 搜索企业 → 进入详情页 → 编辑字段 → 保存 → 查看变更日志
- **成功标准**: 单条编辑耗时 < 2分钟,关键字段变更100%记录

**US-3: 高级组合筛选与模板保存**
- As 产业分析师, I want to 组合多个维度筛选企业并保存为模板, so that 我可以快速复用常用查询
- **场景**: 为领导准备"制造业+百度AI已使用+中高成熟度"的企业报告
- **流程**: 进入高级筛选 → 设置行业、AI场景、百度AI状态、成熟度 → 实时预览数量 → 保存为模板"制造业重点企业" → 下次一键加载
- **成功标准**: 筛选响应时间 < 3秒,模板加载即刻生效

**US-4: Dashboard查看产业分布**
- As 产业分析师, I want to 在Dashboard中看到企业分布和趋势, so that 我可以快速识别重点行业和方向
- **场景**: 每周例会前查看产业总览,准备汇报材料
- **流程**: 登录 → 查看Dashboard总览卡片 → 查看行业/AI场景分布图 → 点击图表区块钻取到企业列表
- **成功标准**: Dashboard加载时间 < 3秒,图表可点击钻取

#### 运营人员

**US-5: 使用筛选模板生成活动名单**
- As 运营人员, I want to 使用预设模板快速筛选企业并导出, so that 我可以高效组织活动
- **场景**: 筹备"智能制造AI应用挑战赛",需要制造业+质检场景+已用AI的企业名单
- **流程**: 选择模板"制造业AI应用企业" → 查看筛选结果 → 调整条件(如增加"质检场景") → 导出Excel
- **成功标准**: 从筛选到导出完成 < 10分钟

**US-6: 为企业添加运营标签**
- As 运营人员, I want to 为企业打上运营标签, so that 我可以追踪参与情况和重点跟进对象
- **场景**: 活动邀约后标记"已邀约",参赛后标记"已参赛",需要重点跟进的标记"重点跟进"
- **流程**: 在列表或详情页 → 选择企业 → 添加标签 → 保存
- **成功标准**: 单条打标 < 10秒,批量打标 < 1分钟

#### 系统管理员

**US-7: 用户与权限管理**
- As 系统管理员, I want to 创建用户并分配角色, so that 不同人员只能访问其职责范围内的功能
- **场景**: 新入职分析师需要账号,运营人员不应编辑企业基础信息
- **流程**: 进入用户管理 → 创建账号 → 选择角色(分析师/运营/管理员) → 启用
- **成功标准**: 创建账号 < 2分钟,权限规则即时生效

**US-8: 字段与枚举配置**
- As 系统管理员, I want to 调整行业、AI场景等枚举列表, so that 系统可以适应业务标准变化
- **场景**: 新增一个行业类别"智慧农业",或新增AI场景"农业病虫害识别"
- **流程**: 进入字段配置 → 选择枚举类型(行业/AI场景) → 添加新值 → 确认变更提示 → 保存
- **成功标准**: 枚举调整 < 5分钟,前端选择控件立即更新

## 4. Functional Scope

### In Scope (MVP)

#### 4.1 企业数据管理 (Priority: P0)

**4.1.1 企业数据模型**
- **企业主表**: 企业名称、统一社会信用代码、地区、行业、成立时间、注册资金、员工数、联系方式
- **融资信息**: 最新融资轮次、融资时间、融资金额
- **AI应用信息**: AI场景(多选)、百度AI使用状态(未使用/试用中/已使用)、AI成熟度(低/中/高)
- **运营信息**: 运营标签(多选)、备注、创建时间、更新时间

**4.1.2 批量导入与更新**
- 提供标准Excel/CSV模板下载
- 支持文件上传和字段映射(自动识别表头)
- 去重规则:基于"企业名称+统一社会信用代码"识别重复
- 冲突处理策略:
  - 显示字段差异对比
  - 支持"覆盖""跳过""字段级合并"三种策略
- 导入结果报告:成功条数、跳过条数、错误明细(可下载)

**4.1.3 单条编辑**
- 企业详情页支持所有字段的编辑
- 必填字段校验:企业名称、统一社会信用代码、行业
- 枚举字段提供下拉选择,多选字段支持标签式选择
- 保存后提示"修改成功",并记录变更日志

**4.1.4 变更日志**
- 记录关键字段的修改历史:谁、什么时间、修改了什么字段、从什么值改为什么值
- 关键字段包括:融资轮次、融资金额、AI场景、百度AI使用状态、成熟度
- 在企业详情页提供"变更日志"Tab,按时间倒序展示

#### 4.2 查询与筛选 (Priority: P0)

**4.2.1 基础搜索**
- 全局搜索框:支持企业名称、统一社会信用代码、地区的模糊搜索
- 搜索结果实时展示,支持跳转到企业详情

**4.2.2 企业列表**
- 列表默认展示:企业名称、行业、地区、员工数、融资轮次、AI场景、百度AI状态、成熟度
- 支持列头排序:按企业名称、成立时间、注册资金、员工数排序
- 支持列显示配置:用户可勾选显示/隐藏列
- 分页展示,每页50条,支持快速跳转

**4.2.3 高级筛选**
- 筛选条件包括:
  - 地区(单选/多选)
  - 行业(单选/多选)
  - 员工数区间(输入框)
  - 注册资金区间(输入框)
  - 成立时间区间(日期选择)
  - 融资轮次(单选/多选)
  - AI场景(多选)
  - 百度AI使用状态(单选/多选)
  - AI成熟度(单选/多选)
  - 运营标签(多选)
- 条件卡片式布局,可添加/删除条件
- 实时预览匹配企业数量
- 支持条件逻辑:同一维度内为"或"关系,不同维度间为"且"关系

**4.2.4 筛选模板**
- 保存当前筛选条件为模板,命名并设置可见范围(仅自己/本部门/全局)
- 筛选模板列表展示:模板名称、创建人、创建时间
- 一键加载模板,支持在加载后继续调整条件
- 支持编辑和删除模板

#### 4.3 Dashboard与可视化 (Priority: P0)

**4.3.1 总览卡片**
- 企业总数
- 具备AI应用的企业数及占比
- 使用百度AI的企业数及占比
- 最近30天新增/更新企业数

**4.3.2 分布图**
- 按行业分布(柱状图):横轴为行业,纵轴为企业数量
- 按AI场景分布(柱状图):横轴为AI场景,纵轴为企业数量
- 按AI成熟度分布(饼图):展示低/中/高成熟度占比
- 所有图表支持鼠标悬停显示数值和占比
- 所有图表支持点击钻取到对应筛选条件的企业列表

**4.3.3 趋势图**
- 最近12个月的新增/更新企业数趋势(折线图)
- 支持按月查看新增企业数和更新企业数

#### 4.4 数据导出 (Priority: P0)

**4.4.1 导出配置**
- 弹窗选择导出字段:勾选需要的字段(或使用预设模板)
- 预设模板包括:"基础信息模板""融资信息模板""AI应用模板""运营活动模板"
- 导出格式:Excel(.xlsx)、CSV(.csv)

**4.4.2 条件继承**
- 默认导出当前筛选条件下的企业子集
- 显示"当前筛选条件下共XX家企业,是否导出?"

**4.4.3 导出记录**
- 记录导出操作:操作人、时间、导出条件、导出条数
- 管理员可查看导出审计日志

#### 4.5 用户与权限 (Priority: P1)

**4.5.1 用户管理**
- 创建用户:用户名、姓名、邮箱、初始密码
- 启用/停用用户
- 密码重置(管理员重置或用户自助修改)

**4.5.2 角色与权限**
- **分析师角色**:
  - 读取:所有企业数据、Dashboard、筛选模板
  - 写入:企业数据编辑、批量导入、变更日志、筛选模板
- **运营人员角色**:
  - 读取:所有企业数据、Dashboard、筛选模板
  - 写入:运营标签、备注、筛选模板(仅自己创建的)
  - 限制:不可编辑企业基础信息和融资信息
- **管理员角色**:
  - 完全权限:用户管理、权限配置、字段配置、所有数据操作

#### 4.6 系统配置与字段管理 (Priority: P1)

**4.6.1 字段配置**
- 枚举维护:行业分类、AI场景、成熟度、融资轮次、运营标签
- 支持添加、编辑、删除枚举值
- 变更前警告:"此变更可能影响已有数据统计和筛选,确认继续?"
- 变更后前端选择控件立即更新

**4.6.2 系统日志**
- 记录关键操作:用户登录、数据导入、批量编辑、导出、配置变更
- 管理员可查看和导出系统日志

### Out of Scope (Post-MVP)

- 自动化外部数据抓取(工商信息、百度AI使用数据)
- 复杂CRM功能(销售线索、跟进记录、商机管理)
- 机器学习推荐(高潜力企业预测)
- 移动端App
- 完全自定义字段类型(仅支持枚举配置)
- 多语言支持
- API开放平台

## 5. User Experience Principles

### 核心体验原则

1. **效率优先** — 常用操作路径 ≤ 3次点击,批量操作优于单条操作
2. **数据可信** — 所有关键操作有确认提示,变更有日志,来源可追溯
3. **灵活筛选** — 条件组合无上限,模板复用降低重复劳动
4. **视觉清晰** — 大数据量下仍保持表格和图表的清晰度,避免信息过载
5. **权限透明** — 用户清楚知道自己能做什么、不能做什么,受限操作有友好提示

### 关键交互流程

**Flow 1: 首次登录与数据导入**
1. 用户登录 → 看到引导页:"欢迎使用,点击开始导入您的第一批企业数据"
2. 点击"下载导入模板" → 下载Excel模板
3. 填充数据后上传 → 系统自动识别字段映射 → 用户确认
4. 系统处理去重 → 显示冲突列表 → 用户选择处理策略(覆盖/跳过/合并)
5. 导入完成 → 显示结果报告:"成功导入XX条,跳过XX条,错误XX条(可下载详情)"
6. 点击"查看Dashboard" → 进入首页,看到企业总数和分布图

**Flow 2: 组合筛选与导出**
1. 用户进入"高级筛选"页面
2. 添加筛选条件:行业=制造业、AI场景包含"质检""预测性维护"、百度AI状态="已使用"、成熟度="中""高"
3. 系统实时显示:"当前条件匹配65家企业"
4. 用户点击"保存为模板",命名"制造业质检重点企业",设置可见范围="本部门"
5. 用户点击"查看结果" → 进入企业列表,显示65家企业
6. 用户点击"导出" → 弹窗选择字段(使用"基础信息+AI应用模板")→ 选择Excel格式 → 确认导出
7. 系统生成文件并提供下载,同时记录导出日志

**Flow 3: Dashboard钻取与详情编辑**
1. 用户登录后进入Dashboard
2. 看到"按行业分布"柱状图,发现"制造业"有120家企业
3. 点击"制造业"柱状条 → 自动跳转到企业列表,筛选条件已设置为"行业=制造业"
4. 用户在列表中点击某家企业"XX智能科技有限公司" → 进入详情页
5. 详情页显示"基础信息""经营与融资""AI应用与百度AI""运营信息"四个Tab
6. 用户切换到"AI应用与百度AI"Tab,修改"百度AI使用状态"从"试用中"改为"已使用"
7. 用户点击"保存" → 系统提示"修改成功",并在"变更日志"Tab中记录此次修改

### UI/UX亮点

- **条件卡片式筛选**:每个筛选条件为独立卡片,可添加/删除,避免复杂表单的信息过载
- **实时预览匹配数量**:筛选条件变化时,立即显示匹配企业数,给用户即时反馈
- **图表可点击钻取**:Dashboard的所有图表区块均可点击,自动带入筛选条件跳转到列表
- **列显示配置**:用户可自定义列表显示哪些字段,适应不同工作场景
- **变更日志时间轴**:企业详情页的变更日志以时间轴形式展示,清晰直观
- **破坏性操作确认**:删除、覆盖导入等操作必须弹出确认框,并提示影响范围

## 6. Technical Considerations

### 技术架构

- **前端**:React + Ant Design,支持复杂表格、筛选器和图表交互
- **后端**:Node.js/Python,提供RESTful API
- **数据库**:PostgreSQL/MySQL,关系型数据库设计
- **部署**:私有云/公有云,支持Docker容器化部署

### 数据模型(简化)

**企业主表(companies)**
- id(主键)、name(企业名称)、credit_code(统一社会信用代码)、region(地区)、industry(行业)、established_date(成立时间)、registered_capital(注册资金)、employee_count(员工数)、contact(联系方式)、created_at、updated_at

**融资信息表(financing)**
- id、company_id(外键)、round(融资轮次)、amount(融资金额)、date(融资时间)

**AI应用信息表(ai_applications)**
- id、company_id(外键)、scenarios(AI场景,JSON数组)、baidu_ai_status(百度AI使用状态)、maturity(成熟度)

**运营标签表(operation_tags)**
- id、company_id(外键)、tag(标签名)、created_by(创建人)、created_at

**变更日志表(change_logs)**
- id、company_id(外键)、field(字段名)、old_value(原值)、new_value(新值)、changed_by(修改人)、changed_at

**筛选模板表(filter_templates)**
- id、name(模板名称)、conditions(筛选条件,JSON)、visibility(可见范围)、created_by、created_at

### 性能与扩展性

- 企业表和关键字段建立索引(name、credit_code、industry、region)
- Dashboard统计数据通过物化视图或Redis缓存加速
- 列表查询支持分页和懒加载
- 应用服务支持横向扩展,多实例负载均衡

### 数据备份与恢复

- 每日全量备份,每小时增量备份
- 支持按时间点恢复,应对误删或错误导入
- 备份文件加密存储

### 安全与隐私

- 用户密码加密存储(bcrypt)
- 所有API请求需JWT Token认证
- 敏感操作(导出、删除)记录审计日志
- 数据库连接和文件存储采用SSL/TLS加密

## 7. Open Questions & Risks

### Open Questions

1. **导入文件大小限制**:单次导入最多支持多少条?是否需要分批导入功能?
   - 建议:单次上限5000条,超过提示分批导入

2. **变更日志保留时长**:是否需要定期归档或删除?
   - 建议:保留2年,超过2年归档到冷存储

3. **筛选模板数量限制**:单个用户最多保存多少个模板?
   - 建议:每人20个,全局模板不限

4. **导出频次限制**:是否需要限制单个用户的导出次数?
   - 建议:每人每天50次,超过需审批

### Risks & Mitigation

**风险1:初始数据质量参差不齐**
- **影响**:导入失败率高,用户体验差
- **缓解**:提供详细的模板填写说明和示例数据,导入前进行严格校验,给出明确的错误提示

**风险2:用户习惯从Excel转到系统的学习曲线**
- **影响**:上线初期使用率低
- **缓解**:提供引导页和操作手册,组织内部培训,设置"快速入门"任务引导

**风险3:字段和维度膨胀导致界面复杂**
- **影响**:筛选和列表页面信息过载,用户找不到重点
- **缓解**:设计合理的字段分组和Tab结构,提供列显示配置和筛选模板,避免一次性展示所有字段

**风险4:性能瓶颈(万级数据)**
- **影响**:查询和Dashboard加载缓慢
- **缓解**:通过索引、缓存和物化视图优化性能,必要时引入Elasticsearch支持复杂查询

## 8. Success Metrics

### 用户行为指标

- 周活跃用户率 ≥ 70%(分析师和运营人员)
- 平均每周导入/更新操作次数 ≥ 5次
- 平均每周筛选操作次数 ≥ 20次
- 筛选模板使用率 ≥ 60%(使用模板的筛选次数/总筛选次数)

### 业务效果指标

- 通过系统筛选出的目标企业参与活动数量较上线前提升 ≥ 30%
- 核心重点企业(业务定义)中,近3个月内至少一次数据更新的比例 ≥ 80%
- 区域产业年度报告所用企业数据中,≥ 90%来自系统

### 技术性能指标

- 单次列表查询(≤2000条)响应时间 < 1秒
- 复杂组合筛选查询响应时间 < 3秒
- Dashboard首屏加载时间(千级数据) < 3秒,(万级数据) < 5秒
- 导入任务失败率 < 5%,且100%有错误报告

### 用户满意度指标

- 用户满意度调查评分 ≥ 4/5(易用性、数据准确性、功能完整性)
- 上线3个月内,用户主动提交的功能优化建议 ≥ 10条
